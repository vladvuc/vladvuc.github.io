Abstract

This thesis explores the application of Hybrid Intelligence (HI) principles in Project Management and Software Engineering systems. It proposes a novel approach utilizing a Hybrid Intelligence ontology to assess the level of HI integration within these systems. This assessment is achieved through a metric called "HI-ness," which evaluates the effectiveness of human-machine collaboration in achieving project goals.

• Project Management is a discipline focused on the initiation, planning, execution, monitoring, controlling, and closure of the project. It uses a structured framework to achieve these objectives within defined constraints of time, scope, budget, estimation, client needs, and other segments. A few of the risks of not having a project appropriately managed in the domain of software engineering are:

  • Poor communication between team members
  • Lack of clear project tasks
  • Inadequate resources
  • Poorly defined roles and responsibilities
  • Scope creep
  • Schedule Risks
With the Hybrid Intelligence approach, we propose to control or completely mitigate these risks and improve the overall project success rate.

• Software engineering is the systematic application of engineering principles to the design, development, testing, and maintenance of software systems. It involves a blend of technical skills (programming languages, algorithms etc.) and engineering practices (requirements analysis, design patterns, version control) to create reliable, efficient software that solves some problem.

• Hybrid Intelligence is an emerging field in Artificial Intelligence that combines the capabilities of human input and artificial agent reasoning to create a synergy between them. Like this, we create intelligent machines in mixed teams towards a shared goal [1]

Resarch Questions

To what extent can the HI system help (Software) Project Management and Software Engineering?

RQ1 → How does the design of an ontology impact the representation of workflows in project management and software engineering within an HI system?

RQ2 → To what extent does the integration of a HI system improve the efficiency and effectiveness of software engineering workflows, as measured by specific metrics

RQ3 → What are the most effective metrics for measuring the "HI-ness" of a HI system ni PM and SE, and how can these metrics be validated?
